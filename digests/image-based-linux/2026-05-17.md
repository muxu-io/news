## Executive Summary
Fedora is addressing recent kernel vulnerabilities, while also dealing with package conflicts stemming from a protobuf update in Rawhide. Silverblue users are encountering issues with Nvidia drivers, Microsoft 365 login, and LibreOffice. Kairos released v4.1.0, which includes fixes for network module detection and GRUB detachment, along with dependency bumps.

## Cross-Project Trends
*   **Kernel Vulnerabilities:** The Fedora project is actively responding to a recent spike in kernel vulnerabilities (DirtyFrag, CopyFail, Fragnesia), emphasizing the importance of robust processes for tracking and patching. This impacts Fedora CoreOS directly.
*   **Dependency Management:** The protobuf transition in Fedora Rawhide highlights challenges in managing library dependencies across different releases, especially when packages share files. Conflicts between `protobuf` and `protobuf3` are causing build failures (FEX) and installation issues. This is a common pain point in rolling releases and highlights the need for careful coordination.

## Critical/Urgent
*   **Fedora CoreOS Security:** Users should update Fedora CoreOS to the latest releases (stable: 44.20260419.3.1, testing: 44.20260510.2.1, next: 44.20260510.1.1) to address "Dirty Frag" and "Copy Fail 2" kernel vulnerabilities.
*   **Silverblue Upgrade Issues:** Silverblue users are reporting failures rebasing to Fedora 44 with problems related to NVIDIA drivers. It may be related to Akmods and package dependencies preventing upgrades. Some users also reported complete boot failure after rebase.
*   **Kinoite Upgrade Issues:** Kinoite users are reporting failures rebasing to Fedora 44 with problems related to NVIDIA drivers. Some users also reported hangs post-login which appears to be triggered by BTRFS errors during the desktop session.
*   **NetworkManager VPNC SELinux Issue:** The combination of NetworkManager, VPNC, and SELinux is reportedly broken in Fedora 43 and 44, requiring users to disable SELinux to connect to VPNs.
