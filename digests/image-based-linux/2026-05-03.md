## Executive Summary

Fedora CoreOS 44 has been released with rebasing and some breaking changes related to SSH key locations and SELinux booleans.  There's a bootc installer failure due to malformed PAX extensions during Anaconda deployment, potentially affecting image builds.  Talos Linux released v1.13.0 and v1.14.0-alpha.0, with significant changes including Clang-built kernel, Container Device Interface, and Environment Configuration Document.

## Cross-Project Trends

*   **Fedora 44 Rebase Impact:** Fedora CoreOS and Fedora Atomic Desktops are rebasing to Fedora 44. This involves changes to SSH key locations (requiring potential updates to Ignition/Butane configurations) and the removal of some SELinux boolean defaults, indicating a potential convergence towards upstream defaults.

## Critical/Urgent

*   **bootc installer failure:** The "malformed pax extension" error reported on the Fedora IoT Discourse is a blocker for creating customized Fedora IoT bootc images. It requires investigation into the compatibility of the installer's ostree/libarchive with modern OCI/Podman layer metadata.
    *   [https://discussion.fedoraproject.org/t/bootc-installer-fails-with-malformed-pax-extension-after-anaconda-deploy/190276](https://discussion.fedoraproject.org/t/bootc-installer-fails-with-malformed-pax-extension-after-anaconda-deploy/190276)
*   **Fedora CoreOS Zincati Update Bug:** A bug in Zincati may cause older updates to be applied even when a newer one has been staged. Users with periodic update strategies should manually trigger updates using `sudo /usr/bin/zincati-update-now`.
    *   [https://github.com/coreos/zincati/issues/928](https://github.com/coreos/zincati/issues/928)
