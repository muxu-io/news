## Technical Discussions

*   **Python Dependency Management:** The introduction of the `%pyproject_patch_dependency` macro offers a standardized way to override Python package dependencies in Fedora RPM packaging. This addresses a common pain point and reduces the fragility of existing workarounds.
    *   [%pyproject_patch_dependency macro now available (provisional)](https://lists.fedoraproject.org/archives/list/devel@lists.fedoraproject.org/message/LGI225OISPPC7GLOWGXIS7W4XMLISNCL/)
*   **ELN SIG and fedora-release**: The discussion around creating a separate `fedora-eln-release` package instead of adding ELN-specific conditionals to the main `fedora-release` package indicates an architectural shift to better manage the diverging needs of ELN and Fedora. This also aims to simplify the `fedora-release` package and provide a more direct upstream for `centos-stream-release`.
