## Executive Summary

This week's updates include a security bump for mesa, clarifications on backporting tests in Yocto, and documentation efforts in `meta-tegra` and Yocto Project. NXP's meta-freescale layer has been updated and contains some fixes, while the meta-arm layer also saw updates to recipes and related CI. There are also several reports of Jetson issues on the forums, including CUDA and camera setup problems.

## Critical/Urgent

-   **Security:** Mesa has a security bump from 26.0.0 to 26.0.1 to address a reported security issue, requiring immediate attention.

## Yocto Core

-   **Testing Policy:** The Yocto Project TSC clarified the policy for backporting ptests and selftests to stable branches, now only acceptable with corresponding bugfixes to minimize intermittent failures.
-   **Recipe Dependencies:** A discussion on the Yocto mailing list covered optimizing sysroot preparation time by fine-tuning build-time dependencies to avoid copying unnecessary files into the sysroot. Francesco Valla suggests removing unwanted components from the sysroot after creation as a workaround but acknowledges it is non-portable and doesn't scale well.
-   **Ptest Runner:**  A patch was merged into ptest-runner to set the `PYTHONUNBUFFERED` environment variable. This resolves timeout handling issues with Python ptests by ensuring unbuffered output.
-   **SBOM/CVE Checking:** New recipes for `sbom-cve-check`, `python3-spdx-python-model`, `python3-hatch-build-scripts`, and `python3-shacl2code` have been added to OE core, enabling automated vulnerability checking.
-    **Ruby Reproducibility:** A patch to fix a reproducibility issue in Ruby documentation generation has been merged.
-   **Systemd Upgrade:** Systemd has been upgraded from 258.1 to 259.1, dropping iptables support in `systemd-networkd` and `systemd-nspawn` in favor of `nftables`.  Musl support is now available but comes with no guarantees of continued support.
-   **PKG_CONFIG Changes:** OE-Core refactored how it sets PKG_CONFIG_LIBDIR and PKG_CONFIG_PATH.  This may impact recipes using these variables directly.

## Platform Updates

### NXP i.MX
-   The `meta-freescale` layer received updates, including a fix for an issue with the i.MX 943 OEI DDR configuration variable name.
-   The `sdcard` image type was dropped from machine configuration files because it does not exist.
-   Improvements were made to `imx-oei.inc` to use PACKAGECONFIG support for ECC and TCM, offering better control over OEI build options.
-   The iMXBOOT\_VARIANTS in `imx95-evk.inc` was updated to include rpmsg.
-   Kernel DTS was updated based on the 6.12.49-2.2.0 release.

### NVIDIA Jetson

-   **meta-tegra Docs**: The `meta-tegra` layer documentation was refactored to use mdbook instead of the wiki, enhancing the structure and maintainability of the documentation. The generated documentation is published to `oe4t.github.io`.
-   **`meta-tegra` DCE Overlay:**  The `TEGRA_DCE_OVERLAY` variable was renamed to `TEGRA_FLASHVAR_DCE_OVERLAY` to ensure the setting is propagated into the flash helper script.
-   **Kernel Patch:** The `nvidia-kernel-oot` recipe got an update to the ASoC patch.
-   **Syntax Fix:** A shell syntax error in `TEGRA_BUILD_GUID` within the `standalone-mm-optee-tegra` recipe has been fixed, resolving build errors.
-   **Jetson Forums:** Several issues were reported on the Jetson forums related to JetPack 6.0 and later versions:
    - A user reported a flashing issue with Jetson Linux 36.4.3 on an NVIDIA Jetson Orin NX.
    - Users are having trouble using GPU with CUDA 12.6.
    - Jetpack 6.2.2 is causing outages with multiple applications on Jetson Orin Nano Super

### Raspberry Pi
- No significant updates for this platform.

### ARM Platforms

-   **Trusted Firmware-A:** The LTS versions of Trusted Firmware-A have been updated in `meta-arm`.
-   **OP-TEE Client:** A patch was added to `optee-client` to make it reusable outside of `meta-arm`.
-   **Quilt-Native Dependency:** A missing dependency on `quilt-native` was added to the `arm/classes` to fix the `do_apply_local_src_patches` task.
-   **Trusted Services** meta-arm is in the process of enabling the fTPM SP and will need the developer to stay updated on it's development
