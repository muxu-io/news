## Executive Summary

This week saw the release of Yocto Project 5.0.16 and 5.3.2 which include several security fixes. Glibc was upgraded to 2.43 in OpenEmbedded core, which introduces some build failures and test issues that are being addressed with recipe updates and workarounds. meta-tegra gained support for additional DTB files for alternate board revisions, while meta-freescale aligned with the virtual-opencl-icd standard. NVIDIA released a number of developer blog posts related to AI and Omniverse.

## Critical/Urgent
- **Security:** Yocto Project 5.0.16 and 5.3.2 releases include important security fixes. Check the release notes for CVE details and apply updates.
- **Build System:** Glibc upgrade to 2.43 in OpenEmbedded core requires recipe updates for `busybox`, `glib-2.0`, `ltp`, `barebox-tools`, `libxcrypt`, and `gcc-runtime`. Apply the listed fixes to avoid build failures.

## Yocto Core

- **Release:** Yocto Project 5.0.16 (scarthgap-5.0.16) is now available. This release breaks Ubuntu 20.04 compatibility due to missing kernel headers. Workarounds are provided in the release notes.
- **Release:** Yocto Project 5.3.2 (whinlatter-5.3.2) is now available with various security fixes.
- **glibc:** OpenEmbedded core upgraded to glibc 2.43. This introduces new features like memory sealing and openat2 support, but also requires recipe updates to address build failures and test issues. (requires scarthgap or later)
    - `busybox`: Applied a fix from Debian bug #1128825 to address `printf` ptest failures.
    - `glib-2.0`: Backported an upstream fix from glib commit 7c837a52 to address `gdatetime` ptest failures.
    - `ltp`: Added a patch to undef HAVE_OPENAT2 in lapi/openat2.h to workaround openat2 build failure.
    - `barebox-tools`: Moved fcntl.h include to after typedef definitions to fix a build failure.
    - `libxcrypt`: Avoided discarded-qualifiers build failure.
    - `gcc-runtime`: Added -Wno-error=discarded-qualifiers to CFLAGS to avoid discarded-qualifiers build failure.
- **gettext:** Upgraded from 0.26 to 1.0.
- **bitbake.conf:** Set PACKAGECONFIG vardepvalue to improve sstate reuse.
- **cargo-c:** Updated from 0.10.20 to 0.10.21.
- **lua:** Upgraded from 5.4.8 to 5.5.0. Added readline as PACKAGECONFIG and made it a build dependency.
- **tcl8:** Fixed `tclConfig.sh` after `UNPACKDIR` change.
- **python3-setuptools:** Dropped Windows launcher executables on non-mingw builds.
- **python3-pip:** Dropped unused Windows distlib launcher templates.
- **gcc-source:** Converted recipe to use `allarch.bbclass`.
- **llvm-project-source:** Converted recipe to use `allarch.bbclass`.
- **selftest/glibc:** Added docstrings to the selftest classes.
- **yocto-uninative:** Updated to 5.1 for glibc 2.43

## Platform Updates

- **meta-tegra:**
  - Added `L4T_UBOOT_EXTLINUX_EXTRA_FDTS` variable to install additional DTB files into `L4T_EXTLINUX_BASEDIR`.
  - Set `TEGRA_FLASHVAR_PMIC_CONFIG` to the correct industrial-specific DTS for industrial modules.
  - Documentation updates: Added edit button to each page and a wiki redirects page.
  - Removed bashism in `setup-nv-boot-control` script.

### NXP i.MX
- **meta-freescale:**
  - Aligned `imx-gpu-viv` and `mali-imx` recipes with `virtual-opencl-icd` standard.
  - Fixed Mesa Vulkan ICD provider name in `imx-base.inc`.
  - Dropped unused NXP BSP overrides from `mesa` recipe.

### Raspberry Pi
- **meta-raspberrypi:** A patch was added to `libpisp` to remove `build_rpath` to fix reproducibility issues.
