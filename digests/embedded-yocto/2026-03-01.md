## Executive Summary

This week's updates include the release of Yocto Project 4.0.33, core updates focusing on improving build reproducibility and graphics stack defaults, and continued efforts to update various recipes across meta-freescale and meta-tegra. NVIDIA has released several blogs around AI models, and Jetson users continue to troubleshoot various issues, particularly related to camera drivers and updating Jetpack without data loss.

## Critical/Urgent

- **Security:** Several CVE statuses have been updated in OE Core. This may require action to investigate and address any newly flagged vulnerabilities.
- **Breaking Change:** meta-freescale-3rdparty is migrating to split-variable format. Users need to migrate from UBOOT_CONFIG.

## Yocto Core

- Yocto Project 4.0.33 (kirkstone-4.0.33) has been released. ([https://lists.yoctoproject.org/g/yocto-announce/message/374](https://lists.yoctoproject.org/g/yocto-announce/message/374))
- **Important:** OE Core now defaults `BB_SIGNATURE_HANDLER` to `OEEquivHash`, enabling hash equivalence for improved build time. This requires reproducible builds for optimal performance.
- OE Core is enabling opengl, ptest, multiarch, wayland, and vulkan DISTRO_FEATURES by default.
- **Impactful:** Fixed an issue in OE Core where the kernel recipe incorrectly depended on initramfs, causing unnecessary rebuilds.
- The previously incorrect dependency in OE-Core has been fixed. meta-tegra is dropping the workaround.
- OE Core has moved systemd to INIT_MANAGER to better support "nodistro".
- OE Core recipe cve-exclusions sets status for 5 CVEs.

## Platform Updates

### NXP i.MX

- meta-freescale: The `linux-imx` recipe upgrade to 6.12.49 resulted in dropping low power voice DTs for imx93.
- meta-freescale-3rdparty: Toradex machine configurations are being migrated from the deprecated legacy `UBOOT_CONFIG` format to the split-variable format. **This is a breaking change and requires updates to your configurations.**

### NVIDIA Jetson

- **meta-tegra**: L4T R36.5.0 is the main focus.
    - Several recipes have been updated for R36.5.0: `gstreamer`, `nvidia-kernel-oot`, `tegra-uefi-prebuilt`, `tegra-storage-layout`, and helper scripts.
    - Recipes have been refactored to use upstream NVIDIA git repositories for `nvidia-kernel-oot` builds.
    - `tegra-kernel.inc` now correctly inherits OE-Core's class, removing the need for a workaround.
    - Adds variables for UEFI builds, giving control over flash variables.
- **Jetson Forum:** Users are reporting issues with camera drivers, CUDA compatibility, and updating JetPack.

## Layer Compatibility

- meta-arm: Dropped walnascar compatibility since it is not tested and supported with whinlatter or later.

## Technical Discussions

- The Yocto Project is seeking contractors for a project to create reference container images, leveraging recent improvements in container image building and testing within the Yocto Project. This is a strategic initiative to expand Yocto's reach beyond embedded systems into the Linux container ecosystem.
