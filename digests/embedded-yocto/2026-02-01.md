## Executive Summary

This week's updates include several recipe upgrades in meta-openembedded, notably `wireshark` which includes a fix for a CVE. `meta-arm` is updating to OP-TEE version 4.9, which will be of interest to those using ARM's reference platforms or the Corstone-1000. The Raspberry Pi Smart Display Module was announced, along with documentation improvements in the Yocto Project.

## Critical/Urgent

- **Security:** The `wireshark` recipe in meta-openembedded has been upgraded to version 4.6.3, which contains a fix for CVE-2025-0962.  This should be prioritized for systems using `wireshark`.

## Yocto Core

- **Documentation:** The bitbake user manual has been updated with improved documentation regarding `bbappend` file matching, clarifying the use of the "``%``" wildcard character and providing further examples. ([PATCH v3](https://lists.yoctoproject.org/g/docs/message/8853))
- **Documentation:**  Several updates to documentation, such as a fix for the `install-buildtools` path and removal of obsolete poky references. See the pull requests for master ([master](https://lists.yoctoproject.org/g/docs/message/8847)), whinlatter ([whinlatter](https://lists.yoctoproject.org/g/docs/message/8846)), and scarthgap ([scarthgap](https://lists.yoctoproject.org/g/docs/message/8845)).
