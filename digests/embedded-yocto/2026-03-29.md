## Executive Summary

This week saw a flurry of activity in the Yocto ecosystem, particularly around package updates and new features for SPDX SBOM generation.  The Kirkstone release received a maintenance update (4.0.34), and standardized git tag formats are being enforced for Yocto Project releases, which may require adjustments to local scripts.  NXP BSPs are upgraded to 6.18.2, and several updates appeared in meta-openembedded.  NVIDIA developers are discussing bringup and stability issues on the Jetson platform, including potential fixes involving Jetpack and the device tree.

## Critical/Urgent

*   **Yocto Project Release Tagging Changes:** All official Yocto Project release tags will consistently match the `yocto-v.v.v` format. Branch-specific tags that duplicate the official tags will be removed from all repositories. Existing milestone tags will be replaced or converted to align with the new standard format. This requires adjustments to local scripts or workflows that rely on the older tag formats. [https://lists.yoctoproject.org/g/yocto/message/66360](https://lists.yoctoproject.org/g/yocto/message/66360)

## Yocto Core

*   **Yocto Project 4.0.34 (kirkstone-4.0.34) Released:**  A new update to the Kirkstone branch is available with associated downloads and release notes. [https://lists.yoctoproject.org/g/yocto-announce/message/380](https://lists.yoctoproject.org/g/yocto-announce/message/380)
*   **Discussion: overlayfs-etc on top of dm-verity:** A thread discusses challenges using `overlayfs-etc` with dm-verity, including issues with initramfs scripts and kernel parameters. A fix for init script calling has been submitted to OE-Core [https://lore.kernel.org/openembedded-core/20260326173432.3286250-1-michael.opdenacker@rootcommit.com/T/#u](https://lore.kernel.org/openembedded-core/20260326173432.3286250-1-michael.opdenacker@rootcommit.com/T/#u)
    *   **Security Note:** A reply to the `overlayfs-etc` thread strongly advises against overlaying the entire `/etc` directory, as it undermines the integrity provided by secure boot. It recommends overlaying only specific files and enforcing integrity protection on the upper layer.
