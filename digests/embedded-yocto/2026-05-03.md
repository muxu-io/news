## Platform Updates

### NXP i.MX

-   **`meta-freescale`:** Fixes a `jsoncpp` runtime dependency issue where prebuilt ISP media server binaries linked against an older version of `jsoncpp` caused runtime linking problems due to SONAME mismatches.  The fix patches the installed binaries to require the correct SONAME. (isp-imx: fix jsoncpp runtime dependency)
-   **`meta-freescale`:** Addresses an issue where integer type macros in the exported IPU UAPI header were breaking userspace builds in `imx-gst1.0-plugin`. The fix maps aliases to internal types to avoid conflicts. (linux-imx-headers: fix ipu.h integer type macros)
-   **`meta-freescale`:** Upgrades `imx-gpu-viv` to version 6.4.11.p4.4, normalizing the OpenCL ICD implementation. (imx-gpu-viv: Bump 6.4.11.p3.2 -> 6.4.11.p4.4)
-   **`meta-freescale`:** Updates `imx-gpu-g2d` to version 6.4.11.p4.4. (imx-gpu-g2d: Bump 6.4.11.p3.2 -> 6.4.11.p4.4)
-   **`meta-freescale`:** Fixes an issue in `imx-oei.inc` where `OEI_DEBUG` and `OEI_DDR_CONFIG` were not handled correctly, and the `EXTRA_OEMAKE` variable had an incorrect machine name. (imx-oei.inc: fix OEI_DEBUG, OEI_DDR_CONFIG and the conditional EXTRA_OEMAKE)
-   **`meta-freescale`:** Sets `SECOEXT_FIRMWARE_NAME` to an empty string by default to prevent build failures in `firmware-ele-imx` when the variable is not explicitly defined. (imx-base.inc: set SECOEXT_FIRMWARE_NAME to empty by default)
