## Executive Summary

This week's updates include a mix of kernel and build system advancements across multiple platforms. Key highlights include the upgrade of the Linux kernel to version 6.18 in meta-tegra for NVIDIA platforms and a fix for a rust segmentation fault when linking against LLVM in Yocto core due to host toolchain version differences. Meta-freescale saw updates to recipes, including ffmpeg and wifi firmware cleanup, and support for imx943 was added. Security enhancements were seen in OpenEmbedded core with upgrades to libpng and python3, addressing CVEs and improving overall security posture.

## Critical/Urgent
- **Security:** OpenEmbedded Core upgraded `libpng` to 1.6.55, addressing CVE-2026-25646, a high-severity heap buffer overflow. Update your recipes to include this version: [https://github.com/openembedded/openembedded-core/commit/bc0075350e2a7c1d99b257877280bdad2c772fa6](https://github.com/openembedded/openembedded-core/commit/bc0075350e2a7c1d99b257877280bdad2c772fa6)

## Yocto Core
- **Build System:** A segmentation fault was occurring in rustc when reusing sstate artifacts built with different host toolchain versions (Yocto #16058). This was resolved by enabling dynamic LLVM linking for rust-native, so rustc links against libLLVM.so instead of static archives: [https://github.com/openembedded/openembedded-core/commit/74ba238ff1ba1e9b612aece1989b828f3a8f8770](https://github.com/openembedded/openembedded-core/commit/74ba238ff1ba1e9b612aece1989b828f3a8f8770)
- **Build System:** The `checklayer` tool would hang indefinitely when a layer caused command failures (YOCTO #16170). This has been fixed by properly handling failure events and adding a timeout mechanism: [https://github.com/openembedded/openembedded-core/commit/a2c6bf582287a56ae241499598d3038b3c3cdb4f](https://github.com/openembedded/openembedded-core/commit/a2c6bf582287a56ae241499598d3038b3c3cdb4f)
- **Recipe Update:**  `blktrace` recipe has been updated to correct SRC_URI: [https://github.com/openembedded/openembedded-core/commit/b290db3fbf557837c85bbff7019ad590c37bb886](https://github.com/openembedded/openembedded-core/commit/b290db3fbf557837c85bbff7019ad590c37bb886)
- **Recipe Update:**  `ifupdown` recipe has been updated to correct upstream version check: [https://github.com/openembedded/openembedded-core/commit/4f79b45b7ae78eaa945cd2e82fab05a5e939b770](https://github.com/openembedded/openembedded-core/commit/4f79b45b7ae78eaa945cd2e82fab05a5e939b770)
- **Recipe Update:** `iproute2` recipe has been updated to correct homepage link: [https://github.com/openembedded/openembedded-core/commit/b63c3507f0d48691269f39d7c0ae43ffe7a6cc67](https://github.com/openembedded/openembedded-core/commit/b63c3507f0d48691269f39d7c0ae43ffe7a6cc67)
- **Recipe Update:** `busybox` recipe has been updated to enable SELinux if DISTRO_FEATURES contains 'selinux': [https://github.com/openembedded/openembedded-core/commit/c544f12073ea712c3d3ce08105d52640a7a322b9](https://github.com/openembedded/openembedded-core/commit/c544f12073ea712c3d3ce08105d52640a7a322b9)
