## Executive Summary

A critical vulnerability in the SAML SSO implementation of Sentry allows account takeover. The popular n8n automation platform is facing multiple vulnerabilities including SQL injection, SSRF, and XSS, requiring immediate patching. CoreDNS also has several vulnerabilities including authentication bypass and denial of service.

## Critical / Actively Exploited

*   **Sentry SAML SSO Account Takeover (GHSA-rcmw-7mc7-3rj7):** A critical vulnerability in Sentry's SAML SSO implementation allows an attacker to take over any user account by using a malicious SAML Identity Provider and another organization on the same Sentry instance. Self-hosted users are only vulnerable if they have more than one organization configured and a malicious user has existing access to modify SSO settings for another organization.
    *   **Affected:** Self-hosted Sentry < 26.4.1 (multi-organization instances), Sentry SaaS (prior to April 2026 deployment)
    *   **Mitigation:** Upgrade self-hosted instances to 26.4.1 or higher. Enable two-factor authentication for all users.

No kernel security vulnerabilities were found in the provided content.
