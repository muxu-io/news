## Executive Summary

Several critical vulnerabilities require immediate attention, including command injection and SSRF bypasses in various open-source projects.  Specifically, command injection in AVideo, and several SSRF vulnerabilities in Budibase and DeepSeek TUI are exploitable without authentication or with low privileges, allowing access to sensitive internal resources.  Also, multiple vulnerabilities were identified in Open WebUI that could result in full account takeover.

## Critical / Actively Exploited

*   **AVideo: OS command injection in `on_publish.php` execAsync via unescaped m3u8 URL (High)**
    *   **CVE:** GHSA-xw67-cg5f-4m2r
    *   **Affected Software:** AVideo
    *   **CVSS:** Not available. Command injection is inherently critical.
    *   **Details:** Classic shell metacharacter injection due to missing `escapeshellarg()` calls.
    *   **Action:**  Patch AVideo.

*   **AVideo's Meet plugin: `uploadRecordedVideo.json.php` derives `users_id` from the uploaded filename and calls passwordless `User->login()`, allowing any caller with the Meet shared secret to obtain a session as arbitrary users including admin (High)**
    *   **CVE:** GHSA-qxvm-r42f-5p8j
    *   **Affected Software:** AVideo Meet plugin
    *   **CVSS:** Not available. Authorization bypass leading to account takeover.
    *   **Details:** The `users_id` to credit the upload to is parsed from the FILENAME the caller controls.
    *   **Action:**  Patch AVideo.

*   **Open WebUI: Stored Cross-Site Scripting (XSS) via insufficient server-side input sanitization in Request class (High)**
    *   **CVE:** GHSA-64rr-pp78-62ww
    *   **Affected Software:** NukeViet CMS
    *   **CVSS:** Not available. Stored XSS can lead to account takeover.
    *   **Details:** The Live plugin's "YouTube-style" view renders the live transmission's stream key into an HTML class attribute by raw `echo`, without `htmlspecialchars()`.
    *   **Action:**  Patch NukeViet CMS.

*   **Vm2 Has a Sandbox Breakout Using Async Generator (Critical)**
    *   **CVE:** GHSA-248r-7h7q-cr24
    *   **Affected Software:** vm2
    *   **CVSS:** Not available. Sandbox breakouts are inherently critical.
    *   **Details:** Sandbox breakout vulnerability allows attackers to write code which can escape from the VM2 sandbox and execute arbitrary commands on the host system.
    *   **Action:** Patch vm2.

*   **Open WebUI: Jupyter code execution works despite `ENABLE_CODE_EXECUTION=false` — feature gate bypassed (High)**
    *   **CVE:** GHSA-482j-2pq6-q5w4
    *   **Affected Software:** Open WebUI
    *   **CVSS:** Not available. Code execution in a container is inherently critical.
    *   **Details:** The `/api/v1/utils/code/execute` endpoint executes arbitrary Python code via Jupyter for any verified user, even when the admin has set `ENABLE_CODE_EXECUTION=false`.
    *   **Action:** Patch Open WebUI.

*   **DeepSeek TUI: run_tests Tool Enables RCE via Malicious Repository Without Approval (Critical)**
    *   **CVE:** GHSA-wx44-2q6h-j6p8
    *   **Affected Software:** DeepSeek TUI
    *   **CVSS:** Not available. Execution of arbitrary code without approval is inherently critical.
    *   **Details:** The `run_tests` tool executes `cargo test` in the workspace with `ApprovalRequirement::Auto`, meaning it runs without any user approval prompt. The `cargo test` will then compile and execute arbitrary code.

*   **Better Auth: OAuth callback accepts mismatched `state` when cookie-backed state storage is used without PKCE (High)**
    *   **CVE:** GHSA-wxw3-q3m9-c3jr

    *   **Subsystem:** nouveau/gsp
    *   **Impact:**  Denial of Service
    *   **Fix Status:** Fixed in multiple kernel versions (6.18.19, 6.19.9, 7.0)
    *   **Details:** These WARN_ONs trigger a lot, and we don't seem to have a plan to fix them, so just drop them, as they are most likely harmless.

*   **CVE-2026-43484: mmc: core: Avoid bitfield RMW for claim/retune flags**
    *   **Subsystem:** mmc: core
    *   **Impact:** Denial of Service
    *   **Fix Status:** Fixed in multiple kernel versions (5.15.203, 6.1.167, 6.6.130, 6.12.78, 6.18.19, 6.19.9, 7.0)
    *   **Details:** Writes to claimed in __mmc_claim_host() or retune_now in mmc_mq_queue_rq() can overwrite other bits when concurrent updates happen in other contexts, triggering spurious WARN_ON(!host->claimed). Convert claimed, can_retune, retune_now and retune_paused to bool to remove shared-word coupling.

*   **CVE-2026-43483: KVM: SVM: Set/clear CR8 write interception when AVIC is (de)activated**
    *   **Subsystem:** KVM: SVM
    *   **Impact:** Denial of Service
    *   **Fix Status:** Fixed in multiple kernel versions (6.1.167, 6.6.130, 6.12.78, 6.18.19, 7.0)
    *   **Details:** KVM leaves the interception enabled after AVIC is activated. If KVM emulates INIT=>WFS while AVIC is deactivated, CR8 will remain intercepted in perpetuity.
