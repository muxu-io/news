## Executive Summary
Multiple critical vulnerabilities in OpenClaw, Zebra, and compressing require immediate patching, especially for deployments with untrusted users or complex configurations. Exploitation of these issues could lead to denial of service, remote code execution, and unauthorized data access. Ensure timely updates to mitigate these risks. Additionally, qmail has a new RCE vulnerability with available exploit code.

## Critical / Actively Exploited

*   **CRITICAL: Zebra Consensus Divergence (GHSA-8m29-fpq5-89jj)**: Zebra versions prior to 4.3.1 are vulnerable to a consensus divergence due to improper validation of sighash hash types in V5 transactions and incorrect use of the canonical hash type for V4 transactions. This could lead to network partitioning and double-spend attacks. Upgrade to Zebra 4.3.1 immediately.
*   **CRITICAL: Zebra rk Identity Point Panic (GHSA-452v-w3gx-72wg)**: Zebra versions prior to 4.3.1 are susceptible to a denial-of-service vulnerability due to a panic in the `orchard` crate when processing transactions with an identity `rk` value. An attacker can crash Zebra nodes by submitting a crafted transaction. Upgrade to Zebra 4.3.1 immediately.
*   **CRITICAL: OpenClaw Feishu webhook and card-action validation failure (GHSA-xh72-v6v9-mwhc)**: OpenClaw versions prior to 2026.4.15 could fail open in Feishu webhook mode without a configured `encryptKey` or when handling malformed card-action callbacks. This could allow unauthenticated webhook or card-action traffic to reach command dispatch. Upgrade to OpenClaw 2026.4.15 immediately.
*   **CRITICAL: compressing Git-Delivered Symlink Poisoning (GHSA-4c3q-x735-j3r5)**: The `compressing` npm package (<= v2.1.0) is vulnerable to arbitrary file writes outside the intended extraction root. This bypasses the patch for CVE-2026-24884 using a Directory Poisoning technique (pre-existing symbolic links).
*   **CRITICAL: Flowise Airtable_Agent Code Injection RCE (GHSA-v38x-c887-992f):** FlowiseAI Flowise version 3.0.13 contains a vulnerability that allows remote attackers to execute arbitrary code due to the lack of proper sandboxing when evaluating an LLM generated python script.
