## Executive Summary

Multiple vulnerabilities in widely used open-source projects including `pyLoad`, `Directus`, `Budibase`, `Roundcube Webmail`, and the Linux Kernel require immediate patching to prevent remote code execution, server-side request forgery, and other critical exploits. The `web3.py` library has a default-enabled SSRF vulnerability which should be mitigated by either disabling CCIP Read or by implementing destination validation and override hooks. Several Linux Kernel CVEs address potential denial-of-service and privilege escalation issues.

## Critical / Actively Exploited

*   **pyLoad: Unprotected `storage_folder` enables arbitrary file write and code execution (Incomplete fix for CVE-2026-33509)** - ([GHSA-4744-96p5-mp2j](https://github.com/advisories/GHSA-4744-96p5-mp2j)) - A non-admin user with `SETTINGS` and `ADD` permissions can redirect downloads to the Flask session store, plant a malicious pickle payload, and trigger RCE via deserialization. The exploit is unauthenticated and requires any HTTP request with the crafted session cookie to trigger deserialization.
*   **pyLoad: Improper Neutralization of Special Elements used in an OS Command** - ([GHSA-w48f-wwwf-f5fr](https://github.com/advisories/GHSA-w48f-wwwf-f5fr)) - A non-admin user with `SETTINGS` permission can modify the `avfile` config option in the `AntiVirus` plugin to achieve RCE. The `ADMIN_ONLY_OPTIONS` protection mechanism is only applied to core config options, not plugin config options.
*   **Directus: Authenticated Users Can Extract Concealed Fields via Aggregate Queries** - ([GHSA-38hg-ww64-rrwc](https://github.com/advisories/GHSA-38hg-ww64-rrwc)) - Aggregate functions applied to `conceal` fields incorrectly return raw database values. Combined with `groupBy`, this allows authenticated users to extract concealed values like API tokens and 2FA secrets, leading to account takeover and 2FA bypass.
*   **Directus: TUS Upload Authorization Bypass Allows Arbitrary File Overwrite** - ([GHSA-qqmv-5p3g-px89](https://github.com/advisories/GHSA-qqmv-5p3g-px89)) - Any authenticated user with basic file upload permissions can overwrite arbitrary existing files by UUID via the TUS resumable upload endpoint. This bypasses row-level permission rules and can lead to data loss and metadata corruption. Workaround: Disable TUS uploads by setting `TUS_ENABLED=false`.
*   **Budibase: Unauthenticated Remote Code Execution via Webhook Trigger and Bash Automation Step** - ([GHSA-fcm4-4pj2-m5hf](https://github.com/advisories/GHSA-fcm4-4pj2-m5hf)) - An unauthenticated attacker can achieve RCE by triggering an automation with a Bash step via the public webhook endpoint, executing as `root` inside the container. Requires an admin to have previously created an automation with a Webhook trigger and Bash step with a trigger field template.
*   **Budibase: Command Injection in Bash Automation Step** - ([GHSA-gjw9-34gf-rp6m](https://github.com/advisories/GHSA-gjw9-34gf-rp6m)) - Users with automation creation/modification access can inject malicious commands via template interpolation. Immediate action: Disable bash automation steps until patched.
*   **BentoML: SSTI via Unsandboxed Jinja2 in Dockerfile Generation** - ([GHSA-v959-cwq9-7hr6](https://github.com/advisories/GHSA-v959-cwq9-7hr6)) - Unsandboxed Jinja2 templating in Dockerfile generation enables RCE when a malicious bento archive is imported and containerized.
*   **BentoML: Command Injection in cloud deployment setup script** - ([GHSA-fgv4-6jr3-jgfw](https://github.com/advisories/GHSA-fgv4-6jr3-jgfw)) - system_packages values from bentofile.yaml are interpolated directly into an apt-get install command during cloud deployment without any quoting, leading to RCE on the CI/CD tier.

Mitigations: Provide destination policy controls such as validation/override hook, private-range blocking, or redirect target checks.
*   **libp2p-rendezvous: Unbounded rendezvous DISCOVER cookies enable remote memory exhaustion** - ([GHSA-v5hw-cv9c-rpg7](https://github.com/advisories/GHSA-v5hw-cv9c-rpg7)) - Unauthenticated peers can exhaust server memory by repeatedly issuing `DISCOVER` requests, as the server stores pagination cookies without bounds.
