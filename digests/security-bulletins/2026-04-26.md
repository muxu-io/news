## Executive Summary

This digest highlights several critical vulnerabilities requiring immediate attention, including remote code execution flaws in LiteLLM, Gemini CLI, and various applications due to insecure deserialization or command injection. Multiple authentication bypasses in Traefik and Kirby could lead to unauthorized access.  Several Linux kernel vulnerabilities have been resolved, and users are advised to update.

## Critical / Actively Exploited

*   **LiteLLM: Authenticated command execution via MCP stdio test endpoints**
    *   CVE: None
    *   Affected Software: LiteLLM versions prior to 1.83.7
    *   Description: Any authenticated user could run arbitrary commands on the host.
    *   Patch: Fixed in 1.83.7. Requires PROXY_ADMIN role for the test endpoints.
    *   Workaround: Block `POST /mcp-rest/test/connection` and `POST /mcp-rest/test/tools/list` at reverse proxy or API gateway.
    *   Source: GitHub Security Advisories
