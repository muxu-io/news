## Executive Summary
Moby received a minor release with bug fixes and HTTP keep-alive improvements for registry connections, potentially improving image pull/push performance. RunC released release candidates for versions 1.5.0 and 1.4.2, with a focus on build improvements, deprecations in favor of `moby/sys/devices`, and bug fixes. BuildKit saw a new release with fixes and performance improvements.

## Cross-Project Trends
Both runc and BuildKit address dependency updates. BuildKit updated its embedded runc version. RunC v1.5.0 deprecates some internal APIs, with the intent to replace it with code from `moby/`. This indicates closer collaboration and code sharing between the projects.

## Critical/Urgent
- A regression in runc v1.3.0 could result in a stuck `runc exec` or `runc run` command when the container process runs for a short time. This is fixed in runc v1.4.2 and users are recommended to upgrade if using runc 1.3.0.

## Project Updates

### Docker / Moby
- **Moby v29.4.0-rc.1:** Bug fixes include addressing unremovable containers, fixing privileged containers losing AppArmor profiles on restart, and improved duplicate container exit handling. HTTP keep-alive enabled for registry connections, improving image pull/push performance. Runc updated to v1.3.5. BuildKit updated to v0.29.0. Malformed IP addresses during daemon startup now handled to prevent crashes. ([https://github.com/moby/moby/releases/tag/docker-v29.4.0-rc.1](https://github.com/moby/moby/releases/tag/docker-v29.4.0-rc.1))
- **Moby client/v0.4.0:** Bug fixes and enhancements include added `//go:fix` inline directives to deprecated functions to help automatically migrating using `go fix`. The client now sets a default User-Agent if none was set. ([https://github.com/moby/moby/releases/tag/client%2Fv0.4.0](https://github.com/moby/moby/releases/tag/client%2Fv0.4.0))
- **Moby api/v1.54.1:** Added `Port.Port()` method to return the port-number as a string.  Fixes handling of unmapped ports (ephemeral ports). ([https://github.com/moby/moby/releases/tag/api%2Fv1.54.1](https://github.com/moby/moby/releases/tag/api%2Fv1.54.1))

### Podman Ecosystem
- Podman discussions highlighted several topics related to rootless containers, including sharing volumes between containers with `userns=auto`, using ephemeral users for enhanced isolation, and slow quadlet startup times. The question of best practices for making a project easy to run with podman was also raised.
- Discussions indicate platform engineers are experimenting with advanced rootless features, volume sharing, and quadlet deployment, and seeking guidance on best practices.

### Container Runtimes
- **runc v1.5.0-rc.2:** Requires Go 1.25+. Pre-opens container root filesystem.  Deprecated `EXTRA_BUILDTAGS` and `libcontainer/devices` (use `github.com/moby/sys/devices` instead). ([https://github.com/opencontainers/runc/releases/tag/v1.5.0-rc.2](https://github.com/opencontainers/runc/releases/tag/v1.5.0-rc.2))
- **runc v1.4.2:** Fixes a regression in runc v1.3.0 which can result in a stuck `runc exec` or `runc run` when the container process runs for a short time. Mount sources that need to be open on the host are now closed earlier during container start, reducing the total amount of used file descriptors. ([https://github.com/opencontainers/runc/releases/tag/v1.4.2](https://github.com/opencontainers/runc/releases/tag/v1.4.2))

### BuildKit
- **BuildKit v0.29.0:** Builtin Dockerfile frontend updated to v1.23.0. Git sources can initialize all files from a Git checkout with commit time in the LLB API for better reproducibility. Various file access operations in Git and HTTP sources have been hardened for improved security. Updated CNI plugins to v1.9.1, QEMU emulator support to v10.2.1, and runc to v1.3.5. ([https://github.com/moby/buildkit/releases/tag/v0.29.0](https://github.com/moby/buildkit/releases/tag/v0.29.0))
- **dockerfile/1.23.0:** Git URLs now accept the `mtime=commit` query parameter to initialize checked-out file timestamps to Git commit time.  Dockerfile can now define `SOURCE_DATE_EPOCH` build-arg in the global scope with a default value. ([https://github.com/moby/buildkit/releases/tag/dockerfile%2F1.23.0](https://github.com/moby/buildkit/releases/tag/dockerfile%2F1.23.0))
