## Project Updates

### containerd

*   **containerd 2.3.0-beta.1:** Released as a beta with new features and improvements. Notably, it's the first LTS release with a planned two-year support window, aligning with the Kubernetes release schedule. Features include:
    *   Filesystem copy transfer types.
    *   Shim bootstrap protocol.
    *   OpenTelemetry trace propagation.
    *   NRI enhancements for passing container user, seccomp policy, rlimits, and sysctl to plugins.
    *   Link: [https://github.com/containerd/containerd/releases/tag/v2.3.0-beta.1](https://github.com/containerd/containerd/releases/tag/v2.3.0-beta.1)
*   **containerd API v1.11.0-beta.1:** Released in alignment with containerd 2.3.0, including:
    *   Filesystem copy transfer types.
    *   Shim bootstrap protocol.
    *   Sandbox API update.
    *   Link: [https://github.com/containerd/containerd/releases/tag/api%2Fv1.11.0-beta.1](https://github.com/containerd/containerd/releases/tag/api%2Fv1.11.0-beta.1)

### Moby / Docker Engine

*   **Moby v29.4.0:** Released with bug fixes, enhancements, and dependency updates.
    *   Improved image pull and push performance via HTTP keep-alive for registry connections. This could reduce network overhead and improve overall container operations.
    *   Fixed privileged containers losing AppArmor profiles after restart.
    *   Updated BuildKit to v0.29.0.
    *   Minimum Go version updated to go1.25.
    *   Link: [https://github.com/moby/moby/releases/tag/docker-v29.4.0](https://github.com/moby/moby/releases/tag/docker-v29.4.0)
*   **Docker Forums:** Reported issues with `nftables` configuration, Mosquitto configuration, and healthcheck detach behavior. These issues may indicate areas needing more robust documentation or default configurations.
