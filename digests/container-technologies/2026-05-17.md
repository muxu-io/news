## Executive Summary

Moby released Docker Engine v29.5.0, including a security fix for a denial-of-service vulnerability related to maliciously crafted images (CVE-2026-32288). Buildah v1.39.9 and Skopeo v1.11.5 patched CVE-2025-49713 and CVE-2026-34986 with updated dependencies. BuildKit v0.30.0 was released introducing "compatibility version" to help maintain reproducible builds across versions, and the ability to fetch git sources by commit SHA.

## Cross-Project Trends

Both Buildah and Skopeo addressed CVE-2026-34986 and CVE-2025-49713 by bumping their dependencies. This highlights a common pattern of addressing vulnerabilities in the container ecosystem through coordinated updates to shared libraries and components.

## Critical/Urgent

*   **CVE-2026-32288 (Docker Engine):** A denial of service vulnerability exists where pulling a maliciously crafted image could cause the daemon to allocate unbounded memory when processing sparse tar archives. *Action: Upgrade to Docker Engine v29.5.0 or later.*
*   **CVE-2025-49713 (Buildah):** Upgrade to v1.39.9 or later.
*   **CVE-2026-34986 (Buildah, Skopeo):** Upgrade to Buildah v1.39.9 and Skopeo v1.11.5 or later.
