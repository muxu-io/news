## Executive Summary

This week's updates highlight several security patches across the container ecosystem, primarily addressing CVE-2026-34986 (Jose library) and CVE-2026-35469 (spdystream). Containerd is preparing for its 2.3 release as an LTS, with interesting features relating to filesystem copy and NRI. Finally, import path changes are coming to the Podman ecosystem to prepare for organizational changes.

## Cross-Project Trends

*   Both Buildah and Skopeo released versions to address CVE-2026-34986, indicating a shared dependency on the vulnerable Jose library.
*   Multiple containerd releases (1.7, 2.0, 2.1, 2.2) addressed CVE-2026-35469, highlighting a widespread vulnerability in older versions of `spdystream`.

## Critical/Urgent

*   **CVE-2026-34986:** Affects Buildah (v1.41.9), Skopeo (v1.18.2, v1.20.1, v1.22.2). Update to patched versions.
*   **CVE-2026-35469:** Affects containerd (v1.7.31, v2.0.8, v2.1.7, v2.2.3).  Upgrade to the latest patch releases.
*   **CVE-2026-33414:** Affects Podman (v5.8.2) when running `podman machine init --image` on Windows with Hyper-V. Update to v5.8.2.

*   **containerd (v1.7.31, v2.0.8, v2.1.7, v2.2.3):** Released with security fix for CVE-2026-35469 (spdystream).
