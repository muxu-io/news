## Executive Summary
Recent developments focus on security patches and bug fixes across the container ecosystem. Buildah, Moby, and Podman have addressed CVEs, while Podman 6 release is delayed to incorporate breaking changes, particularly regarding configuration file handling. Docker Desktop users are experiencing network connectivity issues after Windows sleep, and a user reports slow image listing in Podman.

## Cross-Project Trends
Both Buildah and Moby have released versions addressing CVEs related to security vulnerabilities in underlying components. This indicates a coordinated effort to maintain the security of container build and runtime environments.

## Critical/Urgent
*   **CVE-2026-31431 in Moby (Docker Engine):**  Versions 29.4.2 and 29.4.3 address privilege escalation vulnerabilities. 29.4.2 introduced a regression impacting 32-bit programs, which was resolved in 29.4.3.  Users of 29.4.2 who need to use the workaround provided should use the seccomp profile as described in the release notes and upgrade to 29.4.3 ASAP.
*   **CVE-2026-33414 in Podman:** Version 5.8.2 addresses a vulnerability where `podman machine init --image` on Windows with Hyper-V can execute PowerShell commands from a user-specified image path on the host.  **Action Required:** Windows users with Hyper-V backend should upgrade to Podman 5.8.2 immediately.

### Docker / Moby
*   **Moby v29.4.3:** Fixes regression introduced in v29.4.2 that broke 32-bit programs and i386 images due to excessive seccomp restrictions. [https://github.com/moby/moby/releases/tag/docker-v29.4.3](https://github.com/moby/moby/releases/tag/docker-v29.4.3)
*   **Moby v29.4.2:** Addressed CVE-2026-31431 by blocking AF\_ALG sockets and the socketcall(2) multiplexer in the default seccomp profile, preventing in-container privilege escalation.  Known issues with 32-bit programs, mitigated in 29.4.3 [https://github.com/moby/moby/releases/tag/docker-v29.4.2](https://github.com/moby/moby/releases/tag/docker-v29.4.2)
