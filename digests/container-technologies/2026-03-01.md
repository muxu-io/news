## Executive Summary

Docker Engine released v29.3.0-rc.1 with BuildKit updates and fixes for configuration reloads and prune operations. BuildKit saw multiple pre-releases focused on Dockerfile frontend updates and provenance features. Skopeo v1.9.5 addressed CVE-2025-65637 related to a logrus vulnerability and included other backports and testing improvements.

## Cross-Project Trends

Both Docker and BuildKit saw activity in the last week, especially in release candidates.

## Critical/Urgent

*   **Skopeo v1.9.5**: Addresses **CVE-2025-65637**, a security vulnerability in the logrus dependency. Platform engineers should evaluate and upgrade Skopeo deployments to mitigate this risk.

## Project Updates

### Docker / Moby

*   **Docker v29.3.0-rc.1**: Includes BuildKit v0.28.0-rc2 update, fixes for configuration reloads (systemd notify), prune operation failures, and Windows container issues. Also updates Go runtime to 1.25.7. ([https://github.com/moby/moby/releases/tag/docker-v29.3.0-rc.1](https://github.com/moby/moby/releases/tag/docker-v29.3.0-rc.1))

### BuildKit

*   **Multiple Releases (v0.28.0-rc1, v0.28.0-rc2, dockerfile/1.22.0-rc\*, labs variants)**: Focus on updating the built-in Dockerfile frontend (to v1.22.0-rc2), switching to SLSA v1.0 provenance format, enabling parallel image pushing and cache exporting, and adding new source types for accessing blobs from image registries and OCI layouts. The update to SLSA v1.0 might require adjustments in 3rd party tooling due to a case change in the `InvocationID` field. ([https://github.com/moby/buildkit/releases](https://github.com/moby/buildkit/releases))

### Skopeo Ecosystem

*   **Skopeo v1.9.5**: Addresses CVE-2025-65637. ([https://github.com/containers/skopeo/releases/tag/v1.9.5](https://github.com/containers/skopeo/releases/tag/v1.9.5))
