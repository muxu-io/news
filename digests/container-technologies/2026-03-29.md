## Executive Summary

This week was heavily focused on security fixes in the container ecosystem, with Docker Engine (Moby), BuildKit, and crun all releasing versions to address CVEs. BuildKit also received feature updates related to Git sources and `SOURCE_DATE_EPOCH`. Podman Desktop had a couple of minor releases, and there was community discussion around rootless capabilities, volume management, and kube YAML generation.

## Cross-Project Trends

Both Moby/Docker and crun released updates addressing security vulnerabilities this week, emphasizing the continuous need for security vigilance in the containerization landscape. The BuildKit fixes also have a direct impact on Docker builds.

## Critical/Urgent

*   **CVE-2026-34040 (Docker Engine):** Authorization bypass in AuthZ plugins.  Upgrade immediately if using AuthZ plugins.
*   **CVE-2026-33997 (Docker Engine):** Privilege validation bypass in `docker plugin install`. Upgrade immediately.
*   **CVE-2026-33748 (BuildKit):** Insufficient Git URL validation. Upgrade BuildKit. Affects Docker builds.
*   **CVE-2026-33747 (BuildKit):** Arbitrary file write via untrusted frontend. Upgrade BuildKit. Affects Docker builds.
*   **CVE-2026-30892 (crun):** User ID parsing vulnerability in `crun exec -u`. Upgrade crun.

## Project Updates

### Docker / Moby

*   **v29.3.1:** Released with fixes for multiple security vulnerabilities (CVE-2026-34040, CVE-2026-33997, CVE-2026-33748, CVE-2026-33747), daemon crash, panic with closed stream.  Also includes containerd update to v2.2.2 and Go runtime to 1.25.8. ([https://github.com/moby/moby/releases/tag/docker-v29.3.1](https://github.com/moby/moby/releases/tag/docker-v29.3.1))

### BuildKit

*   **v0.29.0-rc1:** Release candidate includes updated Dockerfile frontend (v1.23.0-rc1), hardening of file access operations in Git and HTTP sources, and several bug fixes. ([https://github.com/moby/buildkit/releases/tag/v0.29.0-rc1](https://github.com/moby/buildkit/releases/tag/v0.29.0-rc1))
*   **v0.28.1:** Released with fixes for Git URL validation vulnerability (CVE-2026-33748), arbitrary file write vulnerability (CVE-2026-33747) and a panic when processing invalid .dockerignore patterns. ([https://github.com/moby/buildkit/releases/tag/v0.28.1](https://github.com/moby/buildkit/releases/tag/v0.28.1))
*   **dockerfile/1.23.0-rc1-labs, dockerfile/1.23.0-rc1:** Git URLs now accept the `mtime=commit` query parameter to initialize checked-out file timestamps to Git commit time. Remote builds using a Git context that define `SOURCE_DATE_EPOCH` automatically default to `mtime=commit`. Dockerfile can now define `SOURCE_DATE_EPOCH` build-arg in the global scope with a default value. ([https://github.com/moby/buildkit/releases/tag/dockerfile%2F1.23.0-rc1](https://github.com/moby/buildkit/releases/tag/dockerfile%2F1.23.0-rc1))

### Podman Ecosystem
